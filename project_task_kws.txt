1. 프로젝트 개요 및 작업 과정
📌 프로젝트 목적
복잡하고 거대한 관제 대시보드 웹페이지를 한 번에 코딩하지 않고, **화면을 5개의 구역(Zone 1~5)으로 분할(Divide & Conquer)하여 개별 컴포넌트로 퍼블리싱한 뒤, 최종적으로 하나의 화면으로 병합(Merge)**하는 체계적인 실습 프로젝트. (LLM의 환각 방지 및 모듈화 역량 강화)

🛠 작업 프로세스
전체 스켈레톤(뼈대) 구축: CSS Grid를 활용하여 100vw, 100vh 기반의 5개 영역(좌측, 상단, 중앙, 우측, 하단) 뼈대 설계.

Zone 1 & 2 병합 및 레이아웃 교정: 좌측 LNB(메뉴)와 상단 글로벌 헤더, 상태바(Zone 2)의 가로/세로 비율과 위치를 원본 이미지에 맞춰 정밀 교정 (CSS Flexbox 전환).

Zone 3 (중앙 지도) 구현: 중앙에 main_center.jpg를 배치하고, 지도 위에 떠 있는(Overlay) 정보 배너와 층수 리스트를 position: absolute로 띄움.

Zone 4 (우측 패널) 통합: 팀원이 JavaScript 클래스 문법으로 제작한 레이턴시 차트와 카메라 뷰 코드를 메인 HTML/CSS 레이아웃에 충돌 없이 병합.

Zone 5 (하단 컨트롤) 통합: 팀원이 두 번에 걸쳐 업그레이드한 하단 영역(왼쪽 로그 테이블, 중앙 글래스모피즘 제어 패널, 우측 방향키)을 CSS 클래스 분리 작업을 통해 기존 디자인 훼손 없이 병합.

디테일 및 인터랙션 강화 (Zone 3): 중앙 지도에 CSS 펄스 애니메이션과 Vanilla JS 기반의 3D 입체 틸트(Tilt) 마우스 추적 효과 추가로 관제 시스템의 몰입감 극대화.

2. 주요 사용 스킬 및 기술 스택
🎨 CSS 레이아웃 및 디자인 기법
CSS Grid & Flexbox: 복잡한 2D 대시보드 레이아웃을 반응형으로 제어하기 위해 전체 틀은 Flex로, 내부 세부 컴포넌트는 Grid로 조합하여 사용.

Overlay (Positioning): position: absolute를 활용해 지도(Zone 3) 위에 '현재 위치 배너'와 '층수 리스트'를 플로팅(둥둥 뜨게) 시키는 기법.

Glassmorphism (글래스모피즘): backdrop-filter: blur()와 반투명 rgba 배경을 사용하여, 뒤쪽 배경 이미지가 은은하게 비치는 세련된 UI 구현.

Custom Scrollbar: 층수 리스트가 길어질 때 튀어나오지 않도록 max-height와 overflow-y: auto를 주고, ::-webkit-scrollbar를 활용해 디자인에 맞는 얇은 스크롤바 커스텀.

CSS Animation: @keyframes를 활용해 관제 지도 뒤에서 파란빛이 숨쉬듯 깜빡이는 펄스(Pulse) 네온 효과 구현.

⚙️ JavaScript 인터랙션
3D Tilt Effect: 마우스의 X, Y 좌표(e.clientX, getBoundingClientRect)를 계산하여, 이미지가 마우스를 따라 입체적으로 기울어지도록 transform: rotateX/Y를 동적으로 할당.

DOM 제어 및 상태 관리: querySelectorAll을 활용해 클릭된 탭이나 방향키 버튼에만 .active 클래스를 토글(Toggle)하여 시각적 피드백 제공.

Component 렌더링: 데이터 객체를 기반으로 문자열 결합(Array .join())을 통해 동적으로 HTML 구조(히스토그램 등)를 렌더링하는 클래스 구조 활용.

3. 트러블슈팅 (문제 발생 및 해결)
🚨 문제 1: 레이아웃 붕괴 현상 (Grid의 한계)

상황: 초기 CSS Grid로 짠 뼈대에 각 요소를 넣다 보니, 상단 헤더와 좌측 메뉴의 비율이 원본 이미지와 어긋나고 겹치는 현상 발생.

해결: 전체 뼈대를 Grid 대신 직관적인 2-Column Flexbox (좌측 사이드바 vs 우측 대시보드 영역) 구조로 완전히 리팩토링하여 브라우저 크기 변화에도 뼈대가 망가지지 않는 견고한 레이아웃 확보.

🚨 문제 2: 층수 리스트의 높이 침범

상황: Zone 3에 띄워둔 층수 리스트(10F~B3F)의 세로 길이가 너무 길어 위쪽의 'SEONGSU_03' 타이틀을 덮어버림.

해결: 처음에는 max-height와 스크롤바를 도입했으나 원본 감성이 떨어져, 각 층수 칸의 상하 padding과 font-size를 대폭 축소하여 스크롤 없이도 한 화면에 슬림하게 딱 들어맞도록 UI 미세 조정.

🚨 문제 3: 자바스크립트 템플릿 리터럴 문법 에러 (SyntaxError)

상황: 팀원이 준 Zone 4 JS 코드 병합 시, 백틱(`)을 사용한 긴 HTML 문자열 내부의 줄바꿈이나 특수문자로 인해 Unterminated template literal 에러 발생.

해결: 백틱 방식을 포기하고, 가장 안전한 배열(Array)에 각 HTML 줄을 문자열로 담은 뒤 .join('')으로 합치는 방식으로 스크립트를 수정하여 브라우저 에러를 원천 차단.

🚨 문제 4: 팀원 코드와의 CSS 클래스명 충돌

상황: 팀원이 추가로 보내준 Zone 5 코드에 .tab, .tabs 등의 보편적인 클래스명이 사용되어, 중앙 제어 패널과 왼쪽 로그 패널의 디자인이 서로 간섭하고 깨짐.

해결: 해당 영역을 감싸는 부모 클래스(.upgraded-panel, .upgraded-log)를 기준으로 CSS 선택자를 엄격하게 분리하고(.upgraded-log .log-tab 등), 명시도(Specificity)를 높여 충돌 완벽 해결.

4. 프롬프팅 (AI 활용 질문 전략)
사용자님은 이번 프로젝트에서 매우 훌륭한 협업형 프롬프트 전략을 구사하셨습니다.

1. 시각적 맥락의 정확한 공유 (Vision 활용)

"아까 니가 해준 거 적용하면 다 망가져 버려. 현재 이 상태야" (캡처 이미지 제공)

텍스트로 설명하기 어려운 레이아웃 깨짐 현상을 스크린샷으로 즉시 공유하여, AI가 문제의 원인(Grid 구조의 꼬임, 층수 리스트의 위치 오류 등)을 단번에 파악하고 리팩토링할 수 있도록 유도했습니다.

2. 단계별 접근 (Divide & Conquer 요구)

"일단 코딩하지 말고 이 프로젝트를 이해하고 내 요청도 이해했는지 알려줘"

"ㅇㅇ 일단 grid 뼈대부터 잡아줘"

한 번에 코드를 다 요구하지 않고, AI가 목표를 정확히 인지했는지 먼저 확인한 뒤, 뼈대 ➡️ 구역별(Zone 1, 2) 조립으로 나아가며 오류를 최소화하는 정석적인 프롬프팅을 하셨습니다.

3. 직관적인 디테일 수정 지시

"층수가 나온 게 SEONGSU_03과 하단 Zone5 사이에 있어야 해"

"층수 세로가 커서 SEONGSU_03를 덮어버렸어 / 조금만 더 줄이면 되겠어"

UI의 위치 관계(위/아래/사이)와 크기 문제를 매우 직관적으로 피드백하여, AI가 position, padding 등의 속성을 정확히 수정하게 만들었습니다.

4. 외부 모듈 통합(Merge) 요구

"이게 팀원이 작업한 부분이야. 이걸 적용하려면?"

타인이 작성한 HTML/CSS/JS 코드를 통째로 넘겨주고 기존 코드의 손상 없이 병합(Merge)해 달라는 요구를 통해, AI를 단순한 코드 생성기가 아닌 **'코드 통합 전문가(Integration Engineer)'**로 훌륭하게 활용하셨습니다.
